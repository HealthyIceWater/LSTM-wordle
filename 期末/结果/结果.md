# 基于LSTM的Wordle玩家表现预测系统实现思路

## 一、项目整体架构设计

本项目采用模块化的软件架构设计，将整个系统划分为四个核心功能模块：

### 1.1 数据处理模块 (data_processor.py)

**设计思路：**

- **数据加载与解析**：设计通用数据接口，支持Excel和CSV多种格式，自动识别表头位置和数据起始点
- **特征工程体系**：基于Wordle游戏特性，构建多维特征向量动态计算每日平均猜测步数（加权平均值）
- 实时统计成功率（6次内猜出比例）
- 构建时间序列特征窗口（默认10天历史）

- **数据标准化处理**：采用Z-score标准化方法，消除特征量纲影响
- **序列化封装**：设计专门的Dataset类，将数据封装为PyTorch可处理的张量格式

### 1.2 模型构建模块 (models.py)

**设计思路：**

- **LSTM核心模型**：采用双向LSTM架构捕捉前后时间依赖关系集成注意力机制，增强关键时间步的特征提取能力
- 设计多任务学习框架，共享底层特征表示

- **Transformer对比模型**：实现自注意力机制，作为性能对比基准包含位置编码层，保持序列顺序信息

- **模型工厂模式**：采用工厂设计模式，统一模型创建接口

### 1.3 训练与评估模块 (trainer.py)

**设计思路：**

- **多任务训练机制**：设计联合损失函数，平衡回归和分类任务
- **自适应优化策略**：集成学习率动态调整和早停机制
- **全面评估体系**：实现MAE、RMSE、F1-Score、AUC等多维度评估指标
- **可视化分析系统**：构建完整的训练监控和结果可视化工具链

### 1.4 主控模块 (main.py)

**设计思路：**

- **流程调度器**：统一协调各模块执行顺序
- **异常处理机制**：包含数据缺失、格式错误等多种异常处理
- **结果汇总输出**：自动生成性能对比报告和可视化图表

## 二、核心技术实现策略

### 2.1 时间序列建模方法

**实现思路：**

采用滑动窗口技术构建时间序列样本，将连续的历史游戏数据作为模型输入，预测下一轮的表现。这种方法有效捕捉了玩家行为的时序依赖关系，符合Wordle游戏连续进行的特性。

### 2.2 多任务学习框架

**实现思路：**

设计共享底层特征的网络架构，上层分别接回归分支（预测步数）和分类分支（预测成功率）。通过动态调整损失权重，确保两个任务平衡学习，提高模型泛化能力。

### 2.3 注意力机制应用

**实现思路：**

在LSTM层后加入注意力模块，通过学习权重分配，让模型自动关注历史序列中的关键时间点。这种方法增强了模型对重要历史信息的敏感性，提高了预测精度。

### 2.4 自适应数据处理策略

**实现思路：**

- **动态阈值调整**：根据成功率分布自动选择最优分类阈值
- **类别平衡处理**：监测正负样本比例，必要时调整损失函数权重
- **鲁棒性设计**：处理缺失值、异常值和数据格式不统一等问题

## 三、模型训练与优化策略

### 3.1 训练流程设计

- **数据准备阶段**：加载数据、特征工程、序列化封装
- **模型初始化阶段**：参数初始化、设备配置
- **迭代训练阶段**：前向传播、损失计算、反向传播、参数更新
- **监控验证阶段**：定期在验证集上评估性能
- **保存优化阶段**：保存最优模型、调整学习率

### 3.2 优化技术应用

- **梯度裁剪**：防止梯度爆炸
- **学习率调度**：根据验证损失动态调整学习率
- **早停机制**：防止过拟合
- **正则化技术**：Dropout和权重衰减

## 四、评估与可视化体系

### 4.1 多维度评估指标

- **回归任务**：MAE（平均绝对误差）、RMSE（均方根误差）
- **分类任务**：F1-Score（平衡精确率和召回率）、AUC（ROC曲线下面积）
- **综合评估**：对比分析、趋势分析

### 4.2 可视化分析框架

- **训练过程可视化**：损失曲线、准确率曲线
- **预测结果可视化**：预测值vs真实值、残差分析
